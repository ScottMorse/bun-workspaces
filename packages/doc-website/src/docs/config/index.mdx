# Configuration
## Workspace Configuration

You can add optional configuration to a workspace in its `package.json` file in  <code>"{WORKSPACE_CONFIG_PACKAGE_JSON_KEY}"</code> or in a separate <code>"{WORKSPACE_CONFIG_FILE_PATH}"</code> file.

### Examples:

#### Separate Config File
<p className="workspace-config-path"><code>path/to/your/workspace/{WORKSPACE_CONFIG_FILE_PATH}</code></p>

You can configure an alias for the workspace that can be referenced instead of the full name in commands such as <a href="/cli#run-script">`run-script`</a>.
In the below example, "myApp" may make a simpler reference to a workspace named `"@my-organization/my-application"`.
