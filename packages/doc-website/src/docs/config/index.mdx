# Configuration
## Workspace Configuration

You can add optional configuration to a workspace in its `package.json` file in  <code>"{WORKSPACE_CONFIG_PACKAGE_JSON_KEY}"</code> 
or in a separate <code>{WORKSPACE_CONFIG_FILE_PATH}</code> file.

See [workspace aliases](/concepts/workspace-aliases) and [script execution order](/concepts/script-execution-order) for more information on the fields available.

#### 2. Within package.json

The same configuration can be added to the workspace's `package.json` file in <code>"{WORKSPACE_CONFIG_PACKAGE_JSON_KEY}"</code>.
<SyntaxHighlighter language="json">
  {JSON.stringify(createPackageJsonExample(exampleWorkspaceConfigSimple), null, 2)}
</SyntaxHighlighter>

Note that if both package.json config and <code>{WORKSPACE_CONFIG_FILE_PATH}</code> are present, 
<code>{WORKSPACE_CONFIG_FILE_PATH}</code> will take precedence. It's recommended to only use one type of configuration within your
project for consistency.
