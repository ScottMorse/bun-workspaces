### Parallel Max Value

When running scripts in parallel, you have control over how many concurrent scripts can run at the same time.

You can pass a number value, a percentage value (e.g. `50%`), or one of the keywords `"auto"`, `"default"`, or `"unbounded"`
to the parallel max option.

#### Value Types

* A plain number sets the limit to the exact number of concurrent scripts. Setting the max to `2` then only allows two scripts to run at the same time.

* A value of `"auto"` (default value) sets the limit to the number of available CPUs, derived from [`os.availableParallelism()`](https://bun.com/reference/node/os/availableParallelism).

* A value of `"unbounded"` allows all scripts to run in parallel, but use this with caution, as it can overwhelm a machine and cause instability.

* A percentage value sets the limit to the percentage of the available CPUs. Setting the max to `50%` then sets the limit to half of the available CPUs.

#### Default Limit

A value of `"default"` explicitly references the default limit, which is equal to `"auto"`.

You can override the default with the environment variable <code>{getUserEnvVarName("parallelMaxDefault")}</code>,
which can be set to any of the valid values listed here.
