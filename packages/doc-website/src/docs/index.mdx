This tool lets you decide the complexity of how you use it. 
To get started, all you need is a normal project using [Bun's native workspaces](https://bun.sh/docs/install/workspaces) feature for nested JavaScript/TypeScript packages.

Think of this as a power suit you can snap onto native workspaces, rather than another monorepo framework.

Start running some [CLI commands](/cli) right away in your repo, or take full advantage of the [scripting API](/api) and its features.

## Installation

You can install the package in your project or simply use `bunx bun-workspaces` to immediately use the latest version of the [CLI](/cli). Installation is required to use the [API](/api).

<SyntaxHighlighter language="bash">
{`# Install to use the API and/or lock your CLI version for your project
bun add --dev bun-workspaces

# Start using the CLI with or without the installation step
bunx bun-workspaces --help`}

</SyntaxHighlighter>

If you use the CLI frequently, you might consider using a shell alias.

<SyntaxHighlighter language="bash">
  {`# Add this to .bashrc, .zshrc, or similar 
# to make it available by default in your shell
alias bw="bunx bun-workspaces"`}
</SyntaxHighlighter>

Examples in these docs will use the alias `bw` for `bunx bun-workspaces` for brevity.
