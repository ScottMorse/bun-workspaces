Normally, a ["script"](/concepts/glossary#script) in bun-workspaces references one of the entries found in a workspace's `"scripts"` field in its `package.json` file.

However, it's possible to run a one-off shell command from a workspace's directory, an "inline script", such as via `bw run "echo 'hello'" --inline` in the CLI.

## Shell

Normally, inline scripts are executed using the [Bun shell](https://bun.com/docs/runtime/shell), a bash-like, cross-platform shell, the same as the utility `import { $ } from 'bun'`.

You can pass `--shell=system` to use the native shell instead. In Unix (Linux, macOS, etc.), `sh -c` is called, while in Windows, `cmd /d /s /c` is called.

You can also set the [root](/config/root) <code>{ENV_VARS_METADATA.scriptShellDefault.rootConfigDefaultsKey}</code> or [environment variable](/config/env-vars) <code>{ENV_VARS_METADATA.scriptShellDefault.envVarName}</code> to `"system"` to use the native shell by default.

## Script Name

Scripts from `package.json` are named by the key in the `"scripts"` field. 

This is used in output logs and [script runtime metadata](/concepts/script-runtime-metadata).

Inline scripts, however, have no name by default, but you can provide a name.
